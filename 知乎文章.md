# 用 Python 调阿里云通义万相2.7，从一句话到一段视频，我搭了个完整的生成工具

> 最近在研究 AI 生成视频，折腾了一段时间，把"文字→图片→视频"这条链路打通了，整个过程踩了不少坑，这里完整记录一下。

---

## 一、为什么要自己搭？

直接用通义万相的网页版不是很方便吗？

确实可以，但我有几个实际需求：

1. **想控制提示词结构**：网页版没法批量测试提示词组合
2. **想要本地存档**：生成的图片和视频直接存在本机，不用手动下载
3. **两步流程太割裂**：先去文生图页面生成，再去视频页面上传，流程断掉了
4. **想做人物一致性**：同一个角色反复出现，需要参考图固定形象

所以干脆自己搭一个，把整个流程串起来。

## 二、最终效果

页面是左右分屏布局：

- **左侧（蓝色区）**：文生图，输入描述词 → 生成图片
- **右侧（紫色区）**：图文生视频，上传/选择首帧图 + 输入描述词 → 生成视频

两个区域完全独立运作，互不干扰，各自有自己的参数设置和结果展示区。

## 三、技术选型

没什么花里胡哨的东西，选了最简单的：

| 层 | 技术 |
|---|---|
| 后端 | Python 3 + Flask 3.0 |

为什么不用 Vue/React？因为这个工具功能相对聚焦，原生 JS 足够了，少一层编译构建的麻烦。

## 四、核心实现：两个 API 的调用方式完全不同

这是最值得说的地方。两个模型的 API 调用模式**截然不同**。

### 文生图 API：同步调用，等结果就行

```python
def generate_image(prompt, size="2K", reference_images=None):
    payload = {
        "model": "wan2.7-image-pro",
        "input": {
            "messages": [{
                "role": "user",
                "content": [{"text": prompt}]
            }]
        },
        "parameters": {
            "size": size,         # 1K / 2K / 4K
            "n": 1,
            "prompt_extend": True
        }
    }
    
    response = requests.post(IMAGE_API_URL, json=payload, headers=headers)
    result = response.json()
    
    # 直接拿图片 URL
    image_url = result["output"]["choices"][0]["message"]["content"][0]["image"]
    return image_url
```

调用一次，等几秒，返回图片 URL，下载保存，完事。非常直接。

### 图生视频 API：必须异步，否则超时

这里有个**容易踩的坑**——视频生成耗时长（几分钟），不能用同步调用，必须加异步请求头：

```python
def create_video_task(image_url, prompt, resolution="720P", duration=5):
    headers = {
        "Authorization": f"Bearer {VIDEO_API_KEY}",
        "Content-Type": "application/json",
        "X-DashScope-Async": "enable"   # ← 关键！必须加这个
    }
    
    payload = {
        "model": "wan2.7-i2v",
        "input": {
            "prompt": prompt,
            "media": [
                {
                    "type": "first_frame",
                    "url": image_url    # 公网可访问的图片地址
                }
            ]
        },
        "parameters": {
            "resolution": resolution,   # 720P / 1080P
            "duration": duration,       # 2-15 秒
            "prompt_extend": True,
            "watermark": False
        }
    }
    
    response = requests.post(VIDEO_API_URL, json=payload, headers=headers)
    task_id = response.json()["output"]["task_id"]
    return task_id  # 只返回任务 ID，不等结果
```

接口立刻返回一个 `task_id`，然后需要轮询查询状态：

```python
def query_task_status(task_id):
    url = f"https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}"
    response = requests.get(url, headers=headers)
    data = response.json()
    
    task_status = data["output"]["task_status"]
    
    if task_status == "SUCCEEDED":
        video_url = data["output"].get("video_url", "")
        return {"status": "SUCCEEDED", "video_url": video_url}
    else:
        return {"status": task_status}
```

前端用定时器每 5 秒调一次查询接口，直到状态变为 `SUCCEEDED`：

```javascript
// 前端轮询逻辑
pollTimer = setInterval(async () => {
    const result = await fetch(`/api/task/${currentTaskId}`).then(r => r.json());
    
    if (result.status === 'SUCCEEDED') {
        clearInterval(pollTimer);
        displayGeneratedVideo(result.filename);   // 显示视频
    } else if (result.status === 'FAILED') {
        clearInterval(pollTimer);
        showToast('生成失败', 'error');
    }
    // PENDING / RUNNING 就继续等
}, 5000);
```

## 五、遇到的最棘手的坑

### 坑 1：本地图片 URL 无法被 API 访问

一开始的设想是：用户在浏览器上传图片 → Flask 保存到本地 → 把 `http://127.0.0.1:5000/images/xxx.png` 传给视频 API。

结果报错：

```json
{
  "code": "InvalidParameter.DataInspection",
  "message": "Unable to download the media resource during the data inspection process."
}
```

原因很直白：**阿里云的服务器当然访问不了你本机的 127.0.0.1**。

解决方案：使用阿里云的 DashScope 临时存储 API，把图片上传到阿里云 OSS，拿到 `oss://` 格式的临时地址，再传给视频 API，并加上特殊请求头：

```python
# 两步上传流程
def upload_file_and_get_temp_url(file_path):
    # 第一步：获取上传凭证
    policy = get_upload_policy(VIDEO_API_KEY, "wan2.7-i2v")
    # 第二步：上传到 OSS
    oss_url = upload_file_to_oss(policy, file_path)  # 返回 oss://...
    return oss_url

# 视频生成时加上解析头
if image_url.startswith('oss://'):
    headers["X-DashScope-OssResourceResolve"] = "enable"
```

---

### 坑 2：视频生成成功，但页面死活不显示

日志里明明看到任务 SUCCEEDED，但前端报 `GET /videos/undefined 404`。

排查了很久，最后发现问题出在后端解析视频 URL 的地方：

```python
# ❌ 错误的字段路径（output 里根本没有 results）
if "results" in output:
    video_url = output["results"][0].get("url", "")

# ✅ 正确的字段路径
video_url = output.get("video_url", "")
```

API 实际返回的结构是 `output.video_url`，不是 `output.results[0].url`。因为这个字段读错了，`video_url` 永远是空字符串，视频也就永远下载不下来，filename 自然是 undefined。

调 API 的时候建议**先打印完整的返回 JSON**，不要凭文档猜字段，文档和实际返回可能有出入。

---

### 坑 3：文生图 API 返回的是阿里云 OSS 图片 URL

文生图成功后会返回一个类似这样的 URL：

```
https://dashscope-result-bj.oss-cn-beijing.aliyuncs.com/xxx/xxx.png
```

这个地址是**公网可访问的**，有效期几小时。在做视频生成时，必须用这个原始 URL，而不是本地存储后的 `/images/xxx.png`。

我一开始把图片保存本地后就丢弃了原始 URL，导致视频生成时又遇到了坑 1 的问题。现在的做法是：图片保存本地用于前端展示，原始 OSS URL 单独保存，视频生成时用原始 URL。

## 六、两个功能完全独立的必要性

一开始做完图片生成后，我顺手在 `displayGeneratedImage()` 函数里加了几行：把生成的图片同步显示到右侧视频区，并自动启用"生成视频"按钮。

```javascript
// ⚠️ 这样做会把两个模块耦合在一起
function displayGeneratedImage(filename, aliUrl) {
    // 左侧展示图片...
    
    // 把生成的图片同步推到右侧
    selectedVideoImage.src = localUrl;   // 跨区操作！
    btnGenerateVideo.disabled = false;   // 跨区操作！
}
```

看起来挺方便，但这带来了一个问题：**右侧的状态完全依赖左侧的操作顺序**。如果用户想直接在右侧上传本地图片，而不先做文生图，"生成视频"按钮就是灰的，无法点击。

后来彻底把两侧分开，各自维护独立的状态变量：

```javascript
// 左侧专用：文生图相关
let currentImageFilename = null;

// 右侧专用：视频生成使用的图片（独立）
let videoSourceImageUrl = null;    // oss:// 或 https://，与左侧无关
```

两侧互不干涉，用户在右侧上传图片、输入 URL，完全不需要先走一遍文生图流程。

## 七、项目结构

整个项目结构很清晰：

```
视频生成/
├── app.py              # Flask 后端，所有 API 路由
├── config.py           # 配置（API Key、路径、参数）

├── requirements.txt    # 3 个依赖
├── templates/
│   └── index.html      # 唯一的页面
├── static/
│   ├── css/style.css   # 样式
│   └── js/main.js      # 前端逻辑（轮询、上传、展示）
├── images/             # 生成的图片（本地存档）
└── videos/             # 生成的视频（本地存档）
```

三个 Python 依赖，没有数据库，没有任务队列，没有 Redis。整个项目 `pip install -r requirements.txt` 然后 `python app.py` 就跑起来了。


**第三步：启动**

```bash
python app.py
```

## 九、总结

做这个工具最大的收获是搞清楚了两类 API 模式的区别：

- **同步 API**（文生图）：发请求 → 等响应 → 拿结果，简单直接
- **异步 API**（视频生成）：发请求 → 拿任务 ID → 轮询状态 → 拿结果，耗时任务的标准做法

另外就是**网络访问边界**的问题——任何需要云端 API 处理的资源（图片），都必须是公网可访问的地址，本地路径行不通。这个道理简单，但真正踩到坑的时候才会印象深刻。

如果你也在研究通义万相的 API 接入，希望这篇文章能帮你少踩几个坑。
